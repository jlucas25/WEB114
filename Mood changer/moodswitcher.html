Questions

Why is textContent safer than innerHTML?
textContent is safer because it does not interpret HTML, preventing script injection attacks.

How are HTML elements connected to JavaScript variables using DOM selection?
HTML elements are selected using methods like document.querySelector() and stored in variables.

What would happen to the text box with id="text" if I added this line: text.textContent = "Hello!"?
The text inside that element would change to "Hello!".
